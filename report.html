Informe Del Proyecto

DIFICULTADES DURANTE LA REALIZACIÓN DE LA ENTREGA

Hubo dificultad en un inicio para desplegar la aplicación web en github pages. Esto debido a que nunca había desplegado una aplicación web en dicha pagina por lo que se estuvo un tiempo intentando hacer funcionar la pagina. Aunque se siguieron las instrucciones en la documentación de github, se desplegaba una pagina en blanco o la pagina lanzaba errores aunque los archivos funcionaran sin problema de manera local. Esto se puede evidenciar en los multiples commits que se hicieron al tratar de agregar, modificar y eliminar el archivo index.html entre otros.
Otra gran dificultad se dio a la hora de empezar a realizar el proyecto y tener una idea de como organizar la pagina. A pesar de las referencias que se dieron, existieron bastantes dudas sobre la forma de proceder y sobre como utilizar algunos de los componentes de bootstrap que no se habían utilizado durante la clase. Esto hizo que se utilizara un buen tiempo en una lluvia de ideas sobre como debía quedar la pagina web.
A la hora de realizar la hoja de vida encontrada en la pestaña Resume de la pagina web, hubieron bastantes dificultades a la hora de desarrollarla. Esto debido a que no tenía una idea clara sobre como organizar la información en el espacio de la pagina web. Junto a esta dificultad se agregaba que me hacia falta entender como emplear y acomodar en el espacio ciertos componentes y utilidades proveidos por bootstrap de manera que la hoja de vida quedara bien diseñada y organizada. Esta tarea fue la que mas tiempo me tomó por las dificultades anteriormente mencionadas.
Una dificultad a la que tuve que enfrentarme en algunas ocasiones fue la de intentar agregar dos columnas contiguas, ambas dentro de una columna contigua a otra columna, y estas dos ultimas adentro de una fila. Esto se debe a que no logré encontrar las medidas correctas para que las columnas pudieran estar contiguas y no una arriba de la otra por falta de espacio. Esto ocasionó que no pudiera organizar algo de información de la manera que quería.

COMPONENTES UTILIZADOS

Entre los componentes utilizados que no se vieron en clase se utilizaron los siguientes: -Se uso un carrousel al inicio de la ventana principal.
-Se utilizo un footer en todas las paginas para mostrar los links en los que me pueden seguir, mi información de contacto y para que las personas interesadas en trabajar conmigo me envien un email.
-Se utilizaron unas cuantas progress bars para mostrar mi progreso utilizando algunas tecnologías en la pestaña de Resume, en la sección de skills.
-Se utilizo un set de cards para presentar la pestaña de experiencia laboral, utilizando 9 cartas en un grid.

A lo largo de todo el proyecto tambien se hizo uso de una navbar para navegar entre pestañas. Filas, columnas y contenedores para ordenar la información y las imagenes presentadas. Tambien labels, textfields y textareas para la sección de contact me. Por ultimo se emplearon algunos botones en la ventana principal para acceder a otras secciones de la pagina.

APORTES Y CONCLUSIONES DE LA ENTREGA

Entre los aportes que me dejo el proyecto. Primero afianzó lo aprendido durante las clases en cuanto a utilización de HTML, css y Bootstrap, dandome una mayor proficiencia de estas herramientas. Segundo, logre aprender a desplegar una pagina web en github pages, valioso aprendizaje que me permitira subir otros proyectos de desarrollo de aplicaciones a esta plataforma en el caso de realizar proyectos pequeños. Tercero, el enfrentarme a este proyecto tambien me ha dado parte de las herramientas necesarias y mas seguridad cuando tenga desarrollar aplicaciones web utilizando librerias como vue.js o React, o frameworks como Angular. Aun mas si empleo bootstrap en conjunto con alguna de estas herramientas. Por ultimo, me enseñó que emplear estrategias de diseño de front-end puede ayudar a facilitar y a agilizar el proceso de implementación de la interfaz grafica, y pude comprobar esto al usar esta estrategia para cada una de las ventanas que implemente despues de la ventana inicial. De esta manera, no tengo que pensar tanto en la forma en la que va a quedar la pagina pues el diseño me da una buena idea de como debería verse.

Entre las conclusiones, debo agregar que emplear HTML, css y Bootstrap para implementar paginas web da lugar a una combinación poderosa de herramientas que permiten la realización de prototipos simples que dan una idea a los desarrolladores, clientes y usuarios sobre como se va a ver e incluso sobre como podría funcionar la pagina web. Tambien puede utilizarse de manera practica para hacer paginas web sencillas como la del ejercicio que guarden, por ejemplo, un portafolio o un curriculum vitae bien presentado de una manera interesante y atractiva, y así que un cliente o desarrollador pueda darse a conocer.
